# Swagger RAG — Vector DB + Claude Code MCP Integration

## Why

The zedcloud repo has 20+ Swagger/OpenAPI spec files (~5.4 MB total) across multiple service
directories. Sending full specs to Claude on every query is expensive and slow. This tooling
sets up a local RAG pipeline: embeddings generated by a local model (zero API cost), stored in
Qdrant, and exposed to Claude Code via an MCP server so Claude can semantically search or fetch
exact endpoint specs on demand.

---

## Stack

| Component | Choice | Reason |
|---|---|---|
| Embedding model | `nomic-embed-text` via Ollama | Native Metal on M3, 274 MB, 768-dim, 8192-token context |
| Vector DB | Qdrant (Docker) | Persistent, filterable, REST API |
| Ingestion | `ingest.py` — Python + `jsonref` | Resolves all `$ref` before chunking (critical for Swagger) |
| MCP server | `server.py` — Python `mcp` library | Stdio transport, spawned on-demand by Claude Code |

## First-Time Setup

### 1. Ollama (embedding model)
```bash
brew install ollama
ollama serve &
ollama pull nomic-embed-text
```

### 2. Qdrant (vector DB)
```bash
docker run -d \
  --name qdrant \
  --restart unless-stopped \
  -p 6333:6333 -p 6334:6334 \
  -v ~/qdrant_storage:/qdrant/storage \
  qdrant/qdrant
# Dashboard: http://localhost:6333/dashboard
```

### 3. Python environment
```bash
cd ~/swagger-rag
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
```

> **Note:** Requires `qdrant-client>=1.18`. This version replaced `client.search()` with
> `client.query_points()`. The code in `server.py` already uses the new API — do not downgrade.

### 4. Ingest swagger files

> **Note:** Files named generically (`swagger.yaml`, `openapi.yaml`) are automatically given a
> source name derived from their parent directory (e.g. `srvs/ganges/api/swagger.yaml` →
> `ganges_swagger`). If you re-ingest after renaming a file, delete the old points from Qdrant
> first to avoid stale entries with the old source name:
> ```python
> from qdrant_client import QdrantClient
> from qdrant_client.models import Filter, FieldCondition, MatchValue
> client = QdrantClient("http://localhost:6333")
> # Delete by old source name, e.g.:
> client.delete("swagger_docs", Filter(must=[FieldCondition(key="source", match=MatchValue(value="swagger"))]))
> ```

```bash
source venv/bin/activate

# Primary service specs
python ingest.py /Users/Varuna_1/git/zedcloud/zservices/swagger/*.swagger.json

# Proto-generated specs (unique services only)
python ingest.py \
  /Users/Varuna_1/git/zedcloud/libs/zmsg/zapiservices/swagger/zedge_app_profile_service.swagger.json \
  /Users/Varuna_1/git/zedcloud/libs/zmsg/zapiservices/swagger/zedge_kubernetes_service.swagger.json \
  /Users/Varuna_1/git/zedcloud/libs/zmsg/zapiservices/swagger/zedge_node_cluster_service.swagger.json \
  /Users/Varuna_1/git/zedcloud/libs/zmsg/zapiservices/swagger/zedge_orchestration_service.swagger.json

# Individual service files
python ingest.py \
  /Users/Varuna_1/git/zedcloud/srvs/ganges/api/swagger.yaml \
  /Users/Varuna_1/git/zedcloud/srvs/indusv2/api/swagger.yaml \
  /Users/Varuna_1/git/zedcloud/srvs/zstore/swagger.yaml \
  /Users/Varuna_1/git/zedcloud/srvs/stream-orchestration/doc/swagger.yaml
```

### 5. Register MCP server with Claude Code

> **Note:** Do not put the server config in `~/.claude/mcp.json` — Claude Code does not read
> that file. Use the `claude mcp add` CLI command instead, which writes to `~/.claude.json`.

```bash
claude mcp add swagger-docs /Users/Varuna_1/swagger-rag/venv/bin/python -- /Users/Varuna_1/swagger-rag/server.py
```

### 6. Open a new Claude Code session

The MCP server is spawned automatically on the next session start. Verify it loaded:
```
claude mcp list   # swagger-docs should show ✓ Connected
```

Points are upserted with stable IDs (`md5(source:method:path)`) so re-running is safe — no duplicates.

```bash
cd ~/swagger-rag && source venv/bin/activate
python ingest.py /Users/Varuna_1/git/zedcloud/zservices/swagger/*.swagger.json
```

## MCP Server Implementation Notes

The `server.py` entry point uses `stdio_server` as an async context manager (required by `mcp>=1.27`):

```python
async def main():
    async with stdio_server() as (read_stream, write_stream):
        await server.run(read_stream, write_stream, server.create_initialization_options())

asyncio.run(main())
```

Do not simplify this to `asyncio.run(stdio_server(server))` — that pattern was removed and will
raise `ValueError: a coroutine was expected`.

---

## Ingestion Design Notes

- **`$ref` resolution**: `jsonref.replace_refs()` inlines all schema references before chunking,
  so each chunk contains the full endpoint spec including request/response schemas.
- **One chunk per endpoint**: each HTTP method + path becomes one vector point.
- **Embedding text** includes: API title, method, path, summary, description, tags, parameters,
  request body description, and response descriptions.
- **Full spec stored** in payload (capped at 6 KB) for retrieval without re-fetching.
- **Context window**: `nomic-embed-text` supports 8192 tokens — large enough for fully inlined
  endpoint specs.
